# 8+16bit 可视化路线建议

状态：后续开发参考  
原则：视觉效果应由真实芯片声道状态驱动，而不是只根据混合音频随机变化。

## 第一阶段视觉目标

```text
中央 CRT 示波器
+ 五声道真实音量条
+ Noise / DPCM 像素粒子
```

这套组合优先使用 Canvas 2D 或 PixiJS，暂不引入完整 Three.js 场景，以控制开发复杂度、内存占用和移动端性能。

## 1. 五声道实时可视化

- [ ] Pulse 1：横向方波能量轨迹（暂时隐藏，等待更低延迟数据）
- [ ] Pulse 2：独立方波能量轨迹（暂时隐藏）
- [ ] Triangle：三角波与低频地形线（暂时隐藏）
- [ ] Noise：碎裂像素、鼓点粒子爆发（暂时隐藏）
- [ ] DPCM：采样脉冲与冲击环（暂时隐藏）
- [x] Mute 后对应动画立即熄灭

已使用 GME 0.6.6 原生 8 路多声道输出取得五个 NES 基础声道的真实
RMS 包络。当前 UI 只显示高刷新率音量条；复杂声道动画因同步观感不佳暂时
隐藏。音量条采用 14 段 LCD 小方格，并在尾部提供最小化 Mute 图标。后续再
增加频率、音高、占空比和寄存器状态。

## 2. 中央示波器

- [x] 显示最终混合音频波形
- 波形和频谱模式切换
- [x] CRT 辉光与网格
- [x] 配色跟随 Theme Kit
- 尊重 `prefers-reduced-motion`
- 页面处于后台时降低或暂停刷新

## 3. 声道特性动画

```text
Pulse 1   方波轨迹
Pulse 2   镜像方波轨迹
Triangle  低频地形线
Noise     像素粒子爆发
DPCM      短促冲击环
```

## 4. 节奏背景

- Noise / DPCM 驱动背景闪光和粒子
- Triangle 控制背景网格起伏
- Pulse 音高控制光线高度或位置
- 避免完全随机、与音乐无关的动画

## 5. 推荐布局

```text
┌──────────── 曲目信息 / 播放控制 ────────────┐
├──────────────────┬─────────────────────────┤
│ 中央示波器        │ 专辑及曲目列表           │
│ 波形 / 频谱       │                         │
├──────────────────┴─────────────────────────┤
│ Pulse1 │ Pulse2 │ Triangle │ Noise │ DPCM │
└────────────────────────────────────────────┘
```

## 6. 实现顺序

1. AudioWorklet 输出整体 RMS 与波形采样
2. 获取五声道独立音量包络
3. Canvas 绘制中央示波器
4. 声道卡片增加动态音量和波形
5. 加入粒子、CRT 余辉和背景网格
6. 性能稳定后再考虑 Three.js 沉浸模式

## 7. 验收原则

- 动画与实际声音同步。
- 五声道不能使用相同的混合 RMS 冒充独立声道数据。
- Mute 后对应视觉立即停止。
- 普通桌面设备目标 60 FPS。
- 移动端降级模式目标 30 FPS。
- 视觉线程不得影响 AudioWorklet 的连续播放。
